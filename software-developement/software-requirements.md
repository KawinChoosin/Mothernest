# Software Requirements

### Project Overview

MotherNest คือแพลตฟอร์มช่วยเหลือคุณแม่มือใหม่แบบครบวงจรในที่เดียว\
ระบบรวมบทความความรู้ การติดตามวัคซีน คอมมูนิตี้ การซื้อสินค้า และการปรึกษาแพทย์

### User Pain Points

#### 1. ข้อมูลกระจัดกระจายและมีมากเกินไป

ในการเป็นคุณแม่มีข้อมูลมากมายที่จำเป็นต้องรู้ อีกทั้งยังต้องหาจากหลายแหล่ง ทำให้ไม่รู้ว่าต้องหาข้อมูลเรื่องไหนเพิ่มอีก

#### 2. ไม่รู้ว่าลูกต้องฉีดวัคซีนอะไรบ้าง

ตั้งแต่แรกเกิด - 12 ปี เด็กต้องฉีดวัคซีนมากกว่า 20 เข็ม และมีเวลาในการฉีดที่แตกต่างกัน

#### 3. ต้องการคำปรึกษา

คุณแม่มือใหม่มักมีคำถามบ่อยครั้งเกี่ยวกับการเลี้ยงดูลูก ทั้งเร่งด่วนและไม่เร่งด่วน



### Software Requirements

### 1. User Roles

* **Guest**: ดูข้อมูลสาธารณะได้บางส่วน
* **Member / Mother**: ใช้งานฟีเจอร์หลักทั้งหมด
* **Doctor**: ให้คำปรึกษาและจัดการเวลานัดหมาย
* **Admin**: จัดการผู้ใช้ เนื้อหา สินค้า รายงาน และระบบ

### 2. Functional Requirements

#### 2.1 Authentication and User Profile

ระบบต้องรองรับการสมัครสมาชิกและเข้าสู่ระบบ\
\- ผู้ใช้ต้องเข้าสู่ระบบได้ด้วยอีเมลหรือเบอร์โทรศัพท์

ระบบต้องรองรับการแก้ไขโปรไฟล์ผู้ใช้\
\- ระบบต้องเก็บข้อมูลพื้นฐาน เช่น ชื่อ วันเกิด เบอร์ติดต่อ และที่อยู่

ระบบต้องรองรับการสร้างโปรไฟล์ลูก\
\- ระบบต้องเก็บข้อมูล(ไม่ละเอียดจนผิด PDPA) เช่น ชื่อเล่น เพศ น้ำหนัก ส่วนสูง และประวัติวัคซีน

#### 2.2 Knowledge Articles

ระบบต้องแสดงบทความความรู้สำหรับคุณแม่มือใหม่\
\- บทความต้องถูกจัดตามหมวดหมู่ เช่น ตั้งครรภ์ หลังคลอด วัคซีน พัฒนาการ และโภชนาการ

ระบบต้องรองรับการค้นหาบทความ\
\- ระบบต้องรองรับการกรองบทความตามหมวดหมู่และคำสำคัญ

ระบบต้องให้ผู้ใช้บันทึกบทความโปรดได้\
\- ระบบต้องแสดงบทความแนะนำตามความสนใจหรือช่วงวัยของลูก

#### 2.3 Vaccine Tracking

ระบบต้องสร้างตารางวัคซีนตามวันเกิดของเด็กได้อัตโนมัติ\
\- ระบบต้องแสดงวัคซีนที่ฉีดแล้ว วัคซีนที่กำลังจะถึง และวัคซีนที่เลยกำหนด

ระบบต้องส่งการแจ้งเตือนก่อนวันนัดวัคซีน\
\- ระบบต้องให้ผู้ใช้บันทึกวันที่ฉีด สถานพยาบาล และหมายเหตุได้

ระบบต้องให้ผู้ใช้ดูประวัติวัคซีนย้อนหลังได้\
\- ระบบควรรองรับการอัปโหลดหลักฐานหรือรูปสมุดวัคซีน

#### 2.4 Community

ระบบต้องรองรับการสร้างโพสต์ในคอมมูนิตี้\
\- ผู้ใช้ต้องสามารถแสดงความคิดเห็น กดถูกใจ และตอบกลับกันได้

ระบบต้องรองรับการตั้งหมวดหมู่โพสต์\
\- ตัวอย่างหมวดหมู่คือ ถาม-ตอบ ประสบการณ์แม่มือใหม่ รีวิวสินค้า และสุขภาพลูก

ระบบต้องมีระบบรายงานโพสต์หรือคอมเมนต์ที่ไม่เหมาะสม\
\- ระบบต้องมีเครื่องมือ moderation สำหรับผู้ดูแลระบบ

#### 2.5 E-Commerce

ระบบต้องแสดงรายการสินค้าสำหรับแม่และเด็ก\
\- ข้อมูลสินค้าต้องมีชื่อ รายละเอียด ราคา รูปภาพ รีวิว และสถานะสต็อก

ระบบต้องรองรับตะกร้าสินค้าและการสั่งซื้อ\
\- ผู้ใช้ต้องเพิ่มสินค้า ลบสินค้า และแก้ไขจำนวนสินค้าได้

ระบบต้องรองรับการชำระเงินออนไลน์\
\- ระบบต้องรองรับการบันทึกที่อยู่จัดส่งและติดตามสถานะคำสั่งซื้อ

ระบบต้องให้ผู้ใช้รีวิวสินค้าได้หลังการซื้อ\
\- ผู้ขายต้องจัดการสินค้า ราคา และสต็อกได้

#### 2.6 Doctor Consultation

ระบบต้องรองรับการจองเวลาปรึกษาแพทย์\
\- ผู้ใช้ต้องเลือกแพทย์ วันที่ เวลา และช่องทางการปรึกษาได้

ระบบต้องรองรับการปรึกษาผ่านแชตหรือวิดีโอคอล\
\- ระบบต้องให้ผู้ใช้ส่งข้อมูลอาการเบื้องต้นก่อนเริ่มปรึกษาได้

ระบบต้องเก็บประวัติการปรึกษา\
\- แพทย์ต้องดูข้อมูลผู้ใช้และข้อมูลลูกที่เกี่ยวข้องได้ตามสิทธิ์

#### 2.7 Notifications

ระบบต้องส่งการแจ้งเตือนสำหรับวัคซีน การนัดแพทย์ คำสั่งซื้อ และกิจกรรมในคอมมูนิตี้\
\- ระบบต้องรองรับการแจ้งเตือนผ่าน in-app notification และ push notification

#### 2.8 Admin Management

ระบบต้องให้ผู้ดูแลจัดการผู้ใช้ บทความ สินค้า และหมวดหมู่ต่าง ๆ ได้\
\- ระบบต้องให้ผู้ดูแลตรวจสอบรายงานปัญหาและดำเนินการกับเนื้อหาที่ไม่เหมาะสมได้

ระบบต้องมี dashboard สำหรับดูสถิติการใช้งานเบื้องต้น\
\- ตัวอย่างคือ จำนวนผู้ใช้ จำนวนคำสั่งซื้อ จำนวนการจองแพทย์ และบทความที่ได้รับความนิยม

### 3. Non-Functional Requirements

#### 3.1 Security

ระบบต้องปกป้องข้อมูลส่วนบุคคลและข้อมูลสุขภาพของผู้ใช้\
\- รหัสผ่านต้องถูกเข้ารหัส และการเข้าถึงข้อมูลต้องเป็นไปตามสิทธิ์ผู้ใช้

ระบบต้องรองรับการยืนยันตัวตนอย่างปลอดภัย\
\- ระบบควรมี audit log สำหรับกิจกรรมสำคัญ

#### 3.2 Performance

ระบบต้องตอบสนองได้รวดเร็วในการใช้งานทั่วไป\
\- หน้าหลัก บทความ และรายการสินค้าควรโหลดได้ภายในเวลาที่เหมาะสม

#### 3.3 Availability

ระบบควรพร้อมใช้งานได้ตลอดเวลา\
\- ฟีเจอร์สำคัญ เช่น การดูวัคซีนและการปรึกษาแพทย์ ต้องมีความเสถียรสูง

#### 3.4 Scalability

ระบบต้องรองรับจำนวนผู้ใช้ที่เพิ่มขึ้นในอนาคต\
\- โครงสร้างระบบต้องขยายได้เมื่อมีผู้ใช้ เนื้อหา สินค้า และข้อมูลสุขภาพเพิ่มขึ้น

#### 3.5 Usability

ระบบต้องใช้งานง่ายสำหรับคุณแม่มือใหม่\
\- หน้าจอต้องเข้าใจง่าย ลดจำนวนขั้นตอน และเหมาะกับการใช้งานบนมือถือ

#### 3.6 Compatibility

ระบบต้องรองรับการใช้งานผ่านมือถือ แท็บเล็ต และเว็บเบราว์เซอร์หลัก\
\- ประสบการณ์ใช้งานควรสม่ำเสมอในทุกอุปกรณ์
