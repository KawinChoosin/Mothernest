# Users Flow

### ประเภทผู้ใช้งาน

<figure><img src="../../.gitbook/assets/image (15).png" alt=""><figcaption></figcaption></figure>

👥**General Users**

* สามารถ log in ผ่าน line OA หรือ web application และเข้าถึงฟังก์ชันหลักของ MotherNest ได้ ได้แก่ **บทความสำหรับคุณแม่ คอมมูนิตี้พูดคุยแบบแม่ๆ Chatbotตอบคำถาม ตลาดขายสินค้า ติดตามวัคซีน**
* หากจ่ายเงินค่าปรึกษากับแพทย์ผู้เชี่ยวชาญเพิ่ม สามารถเข้าถึง**ฟังก์ชันพูดคุยกับแพทย์ผู้เชี่ยวชาญตัวต่อตัว**ได้

**👩🏻‍⚕️Doctors**

* สามารถ log in ผ่าน web application และเข้าถึง**ฟังก์ชันกล่องข้อความ** สามารถพูดคุยกับคนไข้ผ่าน**การพิมพ์ข้อความหรือvideo call** ผ่านกล่องข้อความส่วนตัวของแพทย์และคนไข้ได้

### User Flow - General Users

<figure><img src="../../.gitbook/assets/Screenshot 2026-03-15 005847.png" alt=""><figcaption></figcaption></figure>

#### กรณีเข้าผ่าน Line Official

* หากเป็นผู้ใช้เดิม ผู้ใช้สามารถกดเมนูจาก Rich Menu และเข้าสู่ Web App ตามฟังก์ชันที่เลือกได้ทันที
* หากเป็นผู้ใช้ใหม่ ระบบจะพาเข้าสู่ขั้นตอน `Sign up` ก่อน เพื่อสมัครสมาชิกและยืนยันตัวตนเบื้องต้น

#### กรณีเข้าผ่าน Website Application

* ระบบจะตรวจสอบก่อนว่าเป็นผู้ใช้ใหม่หรือไม่
* หากเป็นผู้ใช้ใหม่ ผู้ใช้ต้องดำเนินการ `Sign up` และ `Create Profile` เพื่อกรอกข้อมูลพื้นฐานที่จำเป็นต่อการใช้งาน
* หากไม่ใช่ผู้ใช้ใหม่ ผู้ใช้สามารถเข้าสู่ `Home Page` ได้ทันที

#### ขั้นตอนหลังสมัครสมาชิก

หลังจากผู้ใช้สมัครสมาชิกเรียบร้อยแล้ว ระบบจะนำไปสู่ขั้นตอน `Create Profile` เพื่อบันทึกข้อมูลส่วนตัวและข้อมูลที่เกี่ยวข้องกับการใช้งาน จากนั้นผู้ใช้จะถูกนำไปยัง `Home Page` ซึ่งเป็นหน้าหลักสำหรับเข้าถึงบริการต่าง ๆ ของระบบ

#### ฟังก์ชันที่ผู้ใช้เข้าถึงได้

เมื่อเข้าสู่ระบบแล้ว ผู้ใช้สามารถเลือกใช้งานฟังก์ชันหลักของ MotherNest ได้ ได้แก่

* ห้องสมุดคอนเทนต์
* AI QA
* พูดคุยแบบแม่ๆ
* ปรึกษาคุณหมอ
* ติดตามวัคซีน
* ร้านค้า

#### สรุปภาพรวม

แนวคิดของ User Flow นี้คือทำให้ผู้ใช้ใหม่เข้าสู่ระบบอย่างเป็นขั้นตอน เริ่มจากการสมัครสมาชิกและสร้างโปรไฟล์ก่อนใช้งานจริง ขณะที่ผู้ใช้เดิมสามารถเข้าถึงบริการที่ต้องการได้รวดเร็วขึ้นผ่านหน้า Home หรือเมนูที่เลือกจาก Line Official

### User Flow - Doctors

<figure><img src="../../.gitbook/assets/image (16).png" alt=""><figcaption></figcaption></figure>

#### ขั้นตอนเริ่มต้นการเข้าใช้งาน

* แพทย์เข้าสู่ระบบผ่าน `Website Application`
* ระบบจะตรวจสอบก่อนว่าเป็นผู้ใช้ใหม่หรือไม่
* หากเป็นผู้ใช้ใหม่ แพทย์ต้องดำเนินการ `Sign up` เพื่อสมัครสมาชิกเข้าสู่ระบบ
* หากไม่ใช่ผู้ใช้ใหม่ แพทย์สามารถเข้าสู่ `Inbox Page` ได้ทันที

#### ขั้นตอนการอนุมัติบัญชีแพทย์

หลังจากสมัครสมาชิกแล้ว ระบบจะมีขั้นตอน `Approve` เพื่อตรวจสอบและอนุมัติสิทธิ์ของแพทย์ก่อนให้ใช้งานฟังก์ชันภายในระบบ

* หากไม่ผ่านการอนุมัติ บัญชีจะอยู่ในสถานะ `Reject`
* หากผ่านการอนุมัติ แพทย์จะเข้าสู่ขั้นตอน `Create Profile`

#### ขั้นตอนการสร้างโปรไฟล์

ในขั้นตอน `Create Profile` แพทย์ต้องกรอกข้อมูลวิชาชีพและข้อมูลส่วนตัวที่จำเป็น เพื่อให้ระบบสามารถแสดงตัวตนของแพทย์และรองรับการเชื่อมต่อกับผู้ใช้งานได้อย่างเหมาะสม หลังจากนั้นระบบจะนำแพทย์ไปยัง `Inbox Page`

#### ฟังก์ชันที่แพทย์เข้าถึงได้

เมื่อเข้าสู่ `Inbox Page` แล้ว แพทย์สามารถใช้งานฟังก์ชันหลักที่เกี่ยวข้องกับการดูแลผู้ใช้ในระบบ ได้แก่

* พูดคุยกับผู้ใช้งานในห้องสนทนาแบบแม่ๆ
* ดูแลและสื่อสารกับผู้ใช้ในส่วน `In-care member`

#### สรุปภาพรวม

User Flow นี้ถูกออกแบบให้มีขั้นตอนตรวจสอบสิทธิ์ก่อนใช้งาน เพื่อให้เฉพาะแพทย์ที่ผ่านการอนุมัติเท่านั้นที่สามารถเข้าถึงระบบได้ ขณะเดียวกันแพทย์ที่ได้รับอนุมัติแล้วจะสามารถเข้าสู่หน้าหลักของการสื่อสารและติดตามผู้ใช้งานได้อย่างเป็นลำดับและชัดเจน
