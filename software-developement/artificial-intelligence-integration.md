# Artificial Intelligence Integration

โครงการนี้นำโมเดลปัญญาประดิษฐ์มาใช้สำหรับการแนะนำบทความที่เหมาะสมกับคุณแม่ตั้งครรภ์แบบเฉพาะบุคคลโดยประเมินจากความสนใจและต้องปลอดภัยสอดคล้องกับอายุครรภ์ทางการแพทย์

### 1) Article Recommendation System 🤖

#### Dataset

* Item Data: ข้อมูลบทความ (เนื้อหา, หมวดหมู่, ช่วงอายุครรภ์เป้าหมาย)
* User Data: ประวัติการเข้าชมบทความของผู้ใช้งาน และอายุครรภ์ปัจจุบัน

#### Model

* ใช้สำหรับจับคู่ความสนใจของคุณแม่กับเนื้อหาบทความได้อย่างแม่นยำ
* ใช้สถาปัตยกรรมแบบ Two-Tower Neural Network
  * พัฒนาด้วยไลบรารี `PyTorch` สำหรับสร้างโครงข่ายประสาทเทียม
  * ใช้ `SentenceTransformer` (โมเดล `all-MiniLM-L6-v2`) สำหรับแปลงข้อความ (Text) เป็นตัวเลข
  * ใช้เทคนิค Shared Embedding แปลงตัวเลขอายุครรภ์เป็นเวกเตอร์ 64 มิติ เพื่อแก้ปัญหาโมเดลติดกับดักสมการเส้นตรง (Linear Trap)
* ข้อจำกัดของการใช้งาน
  * อาจเกิดภาวะ Cold Start กับผู้ใช้ใหม่ที่ยังไม่มีประวัติการอ่าน (ระบบจะสลับไปใช้กฎเกณฑ์ทางการแพทย์ หรือ Rule-based ในการแนะนำแทนชั่วคราว)
*   ตัวชี้วัดประสิทธิภาพ (Metrics)

    * ใช้ `HR@10` (ความแม่นยำ) และ `NDCG@10` (ความสามารถในการจัดอันดับ)

* Item Tower (ฝั่งบทความ): สกัดฟีเจอร์ของบทความ (ชื่อบทความ, หมวดหมู่, อายุครรภ์เป้าหมาย) บีบอัดเป็นเวกเตอร์ขนาด 128 มิติ แล้วบันทึกรอไว้ในฐานข้อมูล

  * Medical Score: นำไปคำนวณผ่านสมการเส้นโค้งระฆังคว่ำ (Gaussian) เพื่อตรวจจับระยะห่างของอายุครรภ์

### 2) RAG Chatbot & Personalization 🤖

นอกเหนือจากระบบแนะนำบทความ (Recommendation System) โครงการ MotherNest ได้บูรณาการระบบ RAG Chatbot (Retrieval-Augmented Generation) เพื่อเป็นผู้ช่วยส่วนตัวที่สามารถโต้ตอบและให้คำปรึกษาคุณแม่ได้อย่างแม่นยำตามหลักการแพทย์

#### 1. RAG Chatbot System (ระบบแชทบอทอัจฉริยะ)

Concept: ใช้สถาปัตยกรรม RAG เพื่อแก้ปัญหาการคิดไปเองของ AI (Hallucination) โดยการบังคับให้โมเดลอ่าน "คู่มือการตั้งครรภ์ที่ผ่านการรับรอง" ก่อนตอบคำถามทุกครั้ง

* Core Technology:
  * LLM Engine: ใช้ Gemini 1.5 Flash บน Vertex AI (เน้นความเร็วและความฉลาดในการทำ Reasoning)
  * Vector Database: ใช้ PostgreSQL (pgvector) บน On-Premise สำหรับเก็บข้อมูลความรู้ทางการแพทย์ที่ละเอียดอ่อน
  * Embedding Model: ใช้ text-multilingual-embedding-002 เพื่อรองรับการค้นหาข้อมูลภาษาไทยและภาษาอังกฤษได้อย่างแม่นยำ
* Workflows (หลักการทำงาน):
  1. Retrieval: เมื่อคุณแม่ถามคำถาม ระบบจะแปลงคำถามเป็นเวกเตอร์แล้วไปค้นหา (Similarity Search) เนื้อหาที่เกี่ยวข้องที่สุดจากฐานข้อมูลคู่มือแม่และเด็กใน Local Storage
  2. Augmentation: นำเนื้อหาที่ค้นพบมาผนวกเข้ากับ "โปรไฟล์ของคุณแม่" (เช่น อายุครรภ์ปัจจุบัน และอาการที่เป็นอยู่)
  3. Generation: ส่งชุดข้อมูลทั้งหมด (Prompt + Context + Profile) ให้ Gemini ประมวลผลเพื่อสร้างคำตอบที่ "เฉพาะบุคคลและปลอดภัยตามอายุครรภ์"

#### 2. Hybrid AI Integration (การทำงานร่วมกันของ AI)

โครงการได้เชื่อมโยงทั้ง 2 ระบบเข้าด้วยกันเพื่อให้เกิดประสบการณ์การใช้งานที่ไร้รอยต่อ:

| **ระบบ**       | **หน้าที่หลัก (Core Function)**              | **การเชื่อมโยงข้อมูล (Data Link)**                         |
| -------------- | -------------------------------------------- | ---------------------------------------------------------- |
| Recommendation | "เชิงรุก" (Proactive) - เสิร์ฟบทความที่ตรงใจ | ใช้ประวัติการอ่านจาก BigQuery มาคำนวณความสนใจ              |
| RAG Chatbot    | "เชิงรับ" (Reactive) - ตอบคำถามข้อสงสัย      | ใช้บริบทจาก Vector Database มายืนยันความถูกต้องทางการแพทย์ |

***

#### 3. Medical Safety Guardrails (กลไกความปลอดภัย)

เพื่อให้มั่นใจว่า AI จะไม่ให้คำแนะนำที่เสี่ยงอันตราย เราได้ติดตั้งระบบตรวจสอบ (Guardrails) ดังนี้:

* Hybrid Scoring for RAG: เช่นเดียวกับระบบแนะนำบทความ แชทบอทจะมีการตรวจสอบ "อายุครรภ์" ในคำตอบ หาก AI แนะนำกิจกรรมที่ไม่เหมาะสมกับไตรมาสปัจจุบัน (เช่น แนะนำการออกกำลังกายหนักในไตรมาสสุดท้าย) ระบบจะทำการ Re-rank หรือแก้ไขคำเตือนทันที
* Emergency Trigger: หากระบบตรวจพบ Keyword วิกฤต (เช่น "เลือดออก", "ปวดท้องรุนแรง") ระบบจะหยุดการตอบด้วย AI และแสดงปุ่ม "โทรสายด่วนโรงพยาบาล" หรือ "ติดต่อแพทย์" ทันทีแทนการให้คำปรึกษาทั่วไป

#### 💡 Architectural Justifications (ทำไมต้องใช้ RAG?)

* Accuracy over Knowledge: เราไม่เชื่อมั่นในความรู้ทั่วไปของ AI (Pre-trained knowledge) เพราะข้อมูลการแพทย์เปลี่ยนแปลงตลอดเวลา การใช้ RAG ทำให้เราสามารถอัปเดต "คู่มือแม่และเด็ก" ในฐานข้อมูลได้ทุกเมื่อ โดยไม่ต้องเทรนโมเดลใหม่
* Privacy First: การทำ RAG บน Hybrid Cloud ช่วยให้เราเก็บ "คลังความรู้เชิงลึก" และ "ข้อมูลส่วนตัวคุณแม่" ไว้ที่ On-Premise (pgvector) และส่งเฉพาะบริบทที่จำเป็นขึ้นไปให้ LLM บน Cloud ประมวลผลเท่านั้น
